# Workflow Spec Recorder操作手册

适用版本：0.2.4  
适用对象：需要把浏览器操作示例、业务资料、输入输出样例转换成 自动化工作流需求文档的用户。

## 1. 软件简介

Workflow Spec Recorder用于记录人工在浏览器和部分 Windows 桌面窗口中的代表性操作，并结合业务资料生成可交付给 自动化工作流生成工具的需求文档。

软件主要提供输出：

- 泛化 APA 需求文档： `APA-generalized-requirements.md，用于描述可动态循环、可泛化执行的业务需求。

## 2. 安装前准备

请先确认电脑满足以下条件：

- 操作系统：Windows。
- 浏览器：Google Chrome，或兼容 Chrome 扩展的 Chromium 浏览器。
- 权限：当前 Windows 用户需要对安装目录和“文档”目录有写入权限。
- 可选：如果要使用大模型生成更完整的需求文档，需要准备 OpenAI-compatible 服务的 Base URL、模型名和 API Key。

说明：

- 不配置 API Key 也可以使用软件，系统会用规则兜底生成 APA 需求文档，但是**不建议**这么使用。
- Chrome 扩展需要手动加载一次，因为 Chrome 不允许普通安装包静默安装本地未上架扩展。

## 3. 安装软件

### 3.1 使用安装包安装

双击安装包按引导步骤完成安装即可

## 4. 首次配置 Chrome 扩展

首次使用前，需要把软件附带的 Chrome 扩展加载到浏览器中。

### 4.1 打开扩展目录

启动桌面端后，点击顶部按钮：

```text
打开插件目录
```

系统会打开扩展所在文件夹。安装包默认位置通常为：

```text
%LOCALAPPDATA%\Programs\WorkflowSpecRecorder\extension
```

免安装发布包中的扩展目录为：

```text
apa-flow-recorder\dist\ApaFlowRecorderSelfContained\extension
```

### 4.2 在 Chrome 中加载扩展

1. 打开 Chrome。

2. 在地址栏输入：
   
   ```text
   chrome://extensions/
   ```

3. 打开右上角“开发者模式”。

4. 点击“加载已解压的扩展程序”。

5. 选择上一步打开的 `extension` 文件夹。

6. 加载成功后，扩展名称为：
   
   ```text
   Workflow Spec Recorder桥接扩展
   ```

### 4.3 检测扩展连接

1. 保持桌面端正在运行。

2. 点击桌面端顶部按钮：
   
   ```text
   检测插件连接
   ```

3. 如果扩展已连接，桌面端会显示“浏览器插件在线”，在线才能使用浏览器录制。

4. 如果未连接，软件会打开本地检测页：
   
   ```text
   http://127.0.0.1:8765/extension-check
   ```

5. 按检测页提示重新加载扩展，或刷新业务页面后，再回到桌面端点击“检测插件连接”。

## 5. 推荐操作流程

1. 启动 `Workflow Spec Recorder`。

2. 点击 `检测插件连接`，确认 Chrome 扩展在线。

3. （可选）在 `录制 / 编辑` 页填写：
   
   - `流程名称`
   - `业务目标`
   - `前置条件`

4. 点击 `开始/继续录制`。

5. 切换到 Chrome，按正常人工方式操作业务系统。

6. 操作过程中，可观察桌面端的实时状态：
   
   - 已记录步骤数
   - 收到浏览器事件数
   - 忽略事件数
   - 最近一次事件是否被记录

7. 如需检查或补充步骤，点击 `暂停`。

8. （可选）在步骤时间线中选择步骤，在右侧编辑业务标题、变量名、成功判定和备注。

9. 确认无误后，点击：导出原始录制包

导出原始录制包后，`workflow.json`、`APA需求文档.md` 和截图资源会自动加入 `泛化需求` 页资料集，无需手动再次选择。

10. 切换到泛化需求页面，生成泛化 APA 需求文档
    
    滑倒最下面。点击 生成 APA 需求文档，等待右侧通知显示已完成，然后在最左侧最下面 点击 打开 泛化导出目录。

11. 打开的目录中有一份 md文档，建议人工阅读检查修改后复制到APA工作区，@这份文档即可让APA开始生成流程

## 6. 生成泛化 APA 需求文档说明

泛化需求用于把“代表性录制示例”和“业务资料”综合起来，生成更适合 自动化工作流生成的需求描述。

录制步骤不等于固定点击脚本。录制只用于说明人工大概如何操作界面、页面上有哪些元素、什么结果算成功。最终需求会尽量抽象出动态循环、输入输出结构、异常处理和约束。

### 6.1 有浏览器操作的流程

1. 先按第 6 章录制几个关键步骤。

2. 点击 `导出原始录制包`，让录制包自动加入资料集。

3. 切换到 `泛化需求` 页。

4. （可选）点击 `添加资料`，继续加入粗略需求、输入样例、输出样例、参考附件或截图说明。

5. 使用大模型，先配置服务商、Base URL、模型名和 API Key，再点击 `生成 APA 需求文档`。

### 6.2 无界面操作的资料处理流程

如果需求属于资料汇总、文档生成、数据清洗、文件转换等无界面操作，可以不录制。

1. 直接切换到 `泛化需求` 页。

2. 点击 `添加资料`，选择需求说明、输入样例、输出样例或参考附件。

3. 在 `补充要求` 中说明希望生成的流程重点。

4. 点击 `生成 APA 需求文档`。

### 6.3 可添加的资料类型

常见可解析资料：

- `.xlsx`
- `.docx`
- `.pptx`
- `.txt`
- `.md`
- `.csv`
- `.json`
- `.xml`
- `.html`
- `.htm`
- `.log`

视觉资料：

- `.png`
- `.jpg`
- `.jpeg`
- `.webp`

补充说明：

- Word 和 PPT 中的内嵌截图、流程图、示意图会被提取为视觉证据。
- 使用支持图像输入的大模型时，图片会和文字说明一起参与分析。
- PDF 当前主要记录文件名和大小。若需要分析 PDF 正文，建议先转成 Word 或 TXT 后再添加。
- 未知类型文件会作为参考附件加入，并记录文件信息。

### 6.4 大模型配置

大模型设置位于 `泛化需求` 页的 `大模型设置（OpenAI-compatible）` 区域。

字段说明：

- `服务商`：便于识别当前配置，例如 `MiniMax` 或 `DeepSeek`。
- `Base URL`：OpenAI-compatible 接口地址。
- `模型名`：要调用的模型名称。
- `API Key`：模型服务访问密钥。

内置预设：

```text
MiniMax 国内预设
Base URL: https://api.minimaxi.com/v1
模型名: MiniMax-M2.7
```

```text
DeepSeek 预设
Base URL: https://api.deepseek.com
模型名: deepseek-chat
```

安全提醒：

- API Key 会明文保存到当前 Windows 用户配置文件。
- 不要把配置文件、导出包、截图或生成的需求文档提交到公共代码仓库。

配置保存位置：

```text
%APPDATA%\ApaFlowRecorder\llm-settings.json
```

### 6.5 提示词模板

提示词模板位于 `泛化需求` 页的 `提示词模板` 区域。

可执行操作：

- 点击 `保存模板` 保存当前模板。
- 点击 `恢复默认` 恢复系统默认模板。
- 点击 `打开模板文件` 在本地编辑器中打开模板文件。

模板文件位置：

```text
%APPDATA%\ApaFlowRecorder\prompt-template.md
```

编辑模板时，建议保留以下占位符：

- `{{recorded_example}}`：注入录制步骤和截图说明。
- `{{source_materials}}`：注入已添加资料内容。
- `{{extra_instruction}}`：注入补充要求。

### 6.6 泛化文档导出位置

默认导出到：

```text
%USERPROFILE%\Documents\ApaFlowRecorder\GeneralizedExports\
```

每次导出会生成一个独立文件夹，文件夹名称格式为：

```text
流程名称-generalized-yyyyMMdd-HHmmss
```

导出文件包括：

- `APA-generalized-requirements.md`：最终 APA 需求文档。
- `workflow-spec.json`：结构化流程规格。
- `llm-prompt.txt`：本次生成使用的提示词。
- `generation-mode.txt`：生成模式说明，例如规则兜底或 LLM 生成。

## 7. 浏览器、下载和桌面操作采集范围

### 7.1 浏览器采集

Chrome 扩展可采集：

- 页面打开和跳转
- 点击
- 文本输入
- 下拉选择
- 文件上传
- 浏览器下载完成或失败
- 页面 URL、标题、DOM 元素、选择器和截图

### 7.2 下载采集

当 Chrome 下载完成或失败时，软件会记录下载步骤，并尽量补充：

- 文件名
- 保存路径
- 来源 URL

### 7.3 桌面采集

录制期间，软件也会尝试采集部分浏览器外操作，例如：

- Windows 保存/打开文件对话框
- 客户端软件窗口
- Excel、ERP 等非浏览器窗口

桌面采集内容包括：

- 桌面点击
- 桌面输入
- Enter、Tab、Escape 等键盘确认或切换动作
- 前台窗口标题
- 进程名
- 控件名称和控件类型

限制说明：

- 密码类控件不会记录明文。
- 少数完全自绘客户端可能只能记录窗口级信息。
- 当前软件只生成需求文档，不自动执行 RPA。

## 8. 推荐工作流

### 8.1 快速生成一份需求文档

1. 启动软件。

2. 检测 Chrome 扩展连接。

3. 填写流程名称、业务目标和前置条件。

4. 点击 `开始/继续录制`。

5. 在 Chrome 中录制 3 到 10 个关键步骤。

6. 回到桌面端点击 `暂停`。

7. 编辑关键步骤的业务标题、变量名和成功判定。

8. 点击 `导出原始录制包`。

9. 切换到 `泛化需求` 页。

10. 添加需求说明、输入样例和输出样例。

11. 点击 `生成 APA 需求文档`。

12. 点击 `打开泛化导出目录`，查看 `APA-generalized-requirements.md`。

### 8.2 只用资料生成需求

1. 启动软件。

2. 切换到 `泛化需求` 页。

3. 点击 `添加资料`。

4. 选择需求说明、样例文件和参考附件。

5. 在 `补充要求` 中写明生成重点。

6. 点击 `生成 APA 需求文档`。

7. 打开泛化导出目录查看结果。

### 8.3 提升生成质量的建议

- 录制时覆盖关键页面、关键按钮、关键输入字段和成功结果页面。
- 给步骤补充清晰的业务标题，而不是只保留“点击”“输入”。
- 对需要动态变化的数据填写变量名。
- 对固定不变的数据保留固定值。
- 在成功判定中说明页面或文件应出现什么结果。
- 添加真实输入样例和目标输出样例。
- 对含有截图、流程图、表格结构的资料，优先使用 Word、PPT、Excel 或图片格式。

## 9. 常见问题处理

### 9.1 点击检测后提示未检测到 Chrome 扩展心跳

处理方式：

1. 确认桌面端正在运行。
2. 确认 Chrome 已加载 `Workflow Spec Recorder桥接扩展`。
3. 在 `chrome://extensions/` 中点击扩展卡片右下角的刷新按钮。
4. 刷新正在操作的业务页面。
5. 点击浏览器右上角扩展图标，确认弹窗显示桌面端连接状态。
6. 回到桌面端再次点击 `检测插件连接`。

### 9.2 开始录制后没有新增步骤

处理方式：

1. 确认桌面端状态为 `录制中`。
2. 确认扩展图标显示 `REC`。
3. 先在普通网页上点击或输入，确认是否能采集。
4. 如果只在浏览器外窗口操作，检查桌面端是否记录了桌面点击或桌面输入。
5. 若仍无步骤，重新加载 Chrome 扩展并重启桌面端。

### 9.3 导出了空原始录制包

可能原因：

- 开始录制前没有点击 `开始/继续录制`。
- Chrome 扩展未连接。
- 录制期间处于暂停状态。
- 操作发生在暂不支持的自绘窗口中。

处理方式：

1. 重新检测插件连接。
2. 确认扩展图标为 `REC`。
3. 重新执行关键操作。
4. 再次导出原始录制包。

### 9.4 生成 APA 需求文档失败

处理方式：

1. 检查添加的资料文件是否仍存在。
2. 如果使用大模型，确认 Base URL、模型名和 API Key 正确。
3. 如果网络或模型服务不可用，先清空 API Key，用规则兜底模式生成。
4. 减少一次性添加的超大资料文件，或把 PDF 正文另存为 Word/TXT。
5. 查看 `generation-mode.txt` 和 `llm-prompt.txt`，辅助定位问题。

### 9.5 生成内容不够贴近业务

优化方式：

1. 在 `补充要求` 中明确目标对象、业务规则、输入输出、异常处理。
2. 添加更完整的输入样例和输出样例。
3. 在录制步骤中补充变量名和成功判定。
4. 添加带截图或流程图的参考资料。
5. 使用支持图像输入的大模型，提高复杂页面理解质量。

### 9.6 找不到导出结果

原始录制包默认路径：

```text
%USERPROFILE%\Documents\ApaFlowRecorder\Exports\
```

泛化需求默认路径：

```text
%USERPROFILE%\Documents\ApaFlowRecorder\GeneralizedExports\
```

也可以在桌面端点击：

- `打开导出目录`
- `打开泛化导出目录`

| 内容          | 默认位置                                                          |
| ----------- | ------------------------------------------------------------- |
| 安装目录        | `%LOCALAPPDATA%\Programs\WorkflowSpecRecorder`                            |

## 11. 卸载软件

可通过以下方式卸载：

1. 打开开始菜单。
2. 找到 `Workflow Spec Recorder` 文件夹。
3. 点击 `卸载 Workflow Spec Recorder`。
4. 按卸载向导完成卸载。

如需同时清理用户数据，可手动检查并删除以下目录中的残留配置、缓存或导出文件：

```text
%APPDATA%\ApaFlowRecorder
%LOCALAPPDATA%\ApaFlowRecorder
%USERPROFILE%\Documents\ApaFlowRecorder
```

删除前请确认其中没有需要保留的导出文档、截图或配置。
